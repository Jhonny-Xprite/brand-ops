# UI Brand Translation

## Product Layer
- tokens globais do BrandOps definem a marca do produto
- dark/light preservam a mesma assinatura emocional

## Client Layer
- `BRAND CORE` injeta overrides por projeto
- shell completa do projeto usa tokens derivados com contraste seguro

## Token Hierarchy
1. base system tokens
2. BrandOps product tokens
3. light/dark tokens
4. project overrides
5. semantic computed tokens

## Fixed Rules
- spacing system nao muda
- arquitetura dos componentes nao muda
- acessibilidade minima nao muda
- feedback semantic nao muda

## Override Rules
- cor
- glow
- hero wash
- sidebar tone
- CTA primario
- badges
- font pairing

## Safety Rules
- cor extrema do cliente nunca quebra contraste
- textos e bordas sao derivados para legibilidade
- hover/focus/error mantem visibilidade
