# Brand Governance

## Product vs Client
- **BrandOps** governa arquitetura, UX, densidade, spacing, comportamento e linguagem do sistema
- **Cliente** governa expressao visual da shell do projeto dentro desses limites

## Product-Owned
- layout
- component architecture
- accessibility
- interaction model
- navigation behavior

## Client-Override
- logo
- wordmark
- simbolo
- cores
- tipografia
- mood visual
- hero/CTA emphasis

## Never Override
- spacing scale
- semantic status meaning
- contrast minimums
- focus visibility
- shell navigation structure

## QA Checklist
- produto e cliente continuam coerentes
- dark/light mantem a mesma identidade
- logo da home sincroniza com o projeto
- nenhuma superficie fica visualmente orfa
