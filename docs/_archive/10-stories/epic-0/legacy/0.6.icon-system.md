# Story 0.6: Icon System Policy

## Status
Done

## Story

**As a** designer and developer,  
**I want** a documented icon policy for the desktop MVP,  
**so that** icon usage stays consistent without over-scoping the design system into a custom icon program

---

## Acceptance Criteria

- [x] A base icon library is selected for MVP use
- [x] The approved icon list covers current workflow needs in Epic 1 and MVP settings/sync flows
- [x] Sizing, stroke, and spacing rules are documented
- [x] Accessibility rules are documented for decorative vs meaningful icons
- [x] Any custom icon need is documented as an exception path, not a default requirement

---

## Tasks / Subtasks

- [x] Evaluate and confirm the base icon library, preferring library-first adoption
- [x] Define icon naming conventions aligned with code usage
- [x] Define approved sizes and placement guidance
- [x] Define status and action icon usage rules
- [x] Document accessibility expectations for icons
- [x] Record the process for requesting a custom icon only if a real gap appears
