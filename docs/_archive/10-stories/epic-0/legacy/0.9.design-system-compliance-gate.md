## Story

**As a** product owner and QA team,  
**I want** a formal compliance gate proving that the current MVP UI now follows the approved design-system,  
**so that** Epic 0 can be considered applied in practice and not only documented in theory

---

## Acceptance Criteria

- [ ] Every surface audited in Story 0.7 receives a final status of `compliant`, `accepted exception`, or `follow-up required`
- [ ] Every accepted exception includes an explicit justification tied to an approved project artifact
- [ ] QA verifies token usage, semantic classes, reusable component extraction, layout alignment, accessibility alignment, and icon-policy alignment
- [ ] The compliance result is documented in a reusable repo-first checklist or QA artifact
- [ ] The story fails if significant drift remains between the approved Epic 0 guides and the actual MVP UI

---

## Tasks / Subtasks

- [ ] Re-run the Story 0.7 audit matrix against the remediated code from Story 0.8
- [ ] Assign a final compliance status to each audited surface
- [ ] Record justified exceptions with explicit artifact traceability
- [ ] Validate reusable extractions against the approved handoff and component-library guides
- [ ] Validate layout, accessibility, and icon-policy adherence
- [ ] Publish a reusable compliance checklist or QA evidence artifact for future UI stories
- [ ] Produce the formal gate decision for PO closeout

[Source: docs/guides/icon-system-policy.md]
- Icon policy adherence is part of compliance where icons appear in the current UI

### Gate Rules

- This story is a QA and governance gate, not a broad implementation story
- Exceptions are allowed only when tied to an approved artifact and documented explicitly
- A green lint/typecheck/test run is necessary but not sufficient on its own
- The gate must confirm practical adoption of the design-system across the current MVP UI
