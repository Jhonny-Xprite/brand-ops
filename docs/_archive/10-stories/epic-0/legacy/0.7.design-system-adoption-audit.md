## Story

**As a** product and implementation team,  
**I want** a full audit of where the approved design-system is and is not being applied in the current MVP UI,  
**so that** remediation work can target real gaps instead of relying on assumptions

---

## Acceptance Criteria

- [x] Every currently implemented MVP UI surface is classified as `aligned`, `partial`, or `non-compliant`
- [x] Each gap is mapped to the correct design-system rule, token, semantic class, or approved component pattern
- [x] The audit separates issues by category: semantic token usage, reusable component absence, layout shell drift, accessibility drift, and icon-policy drift
- [x] The audit includes a remediation priority order that can drive implementation without additional discovery work
- [x] The audit is documented as a repo-first artifact and references the approved Epic 0 guides as source of truth

---

## Tasks / Subtasks

- [x] Inventory all currently implemented MVP UI surfaces in `src/pages/` and `src/components/`
- [x] Classify each surface as `aligned`, `partial`, or `non-compliant`
- [x] Record every deviation with the expected semantic pattern or guide reference
- [x] Separate gaps into adoption categories: token, reusable component, layout, accessibility, and iconography
- [x] Publish the audit matrix in a repo-first guide for UX, Dev, QA, and PO use
- [x] Produce a remediation priority list that feeds Story 0.8 directly

### Current-State Findings to Confirm

- `src/pages/index.tsx` still contains ad hoc `gray/white` styling outside the semantic design-system baseline
- `src/pages/creative-library.tsx` is partially aligned but still contains repeated inline visual patterns that should be governed more explicitly
- `src/components/CreativeLibrary/*` mixes semantic classes with locally duplicated structures for status surfaces, dialogs, and action groups
- Atomic folders in `src/components/{atoms,molecules,organisms}` are underused relative to the repeated UI patterns already present in the codebase

### Audit Scope Rules

- Audit only currently implemented MVP UI surfaces
- Do not invent future screens or component families not already represented in product artifacts
- Treat the approved Epic 0 guides as the decision authority when classifying drift
- The deliverable is an audit artifact, not a remediation implementation
