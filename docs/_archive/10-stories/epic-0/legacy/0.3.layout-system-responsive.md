## Story

**As a** designer and developer,  
**I want** desktop layout rules for the MVP shell and core work surfaces,  
**so that** the app behaves predictably across desktop window sizes without adding mobile-first scope

---

## Acceptance Criteria

- [x] Desktop layout rules are documented for compact, standard, and wide windows
- [x] Sidebar, main workspace, and detail panel patterns are defined
- [x] Overflow, sticky behavior, and table/grid containment rules are documented
- [x] Layout guidance covers library browsing, metadata editing, dashboard, sync, and settings contexts
- [x] The story does not require mobile-first, tablet, or touch-first acceptance criteria

---

## Tasks / Subtasks

- [x] Define desktop window ranges and naming
- [x] Design sidebar + main shell pattern
- [x] Design sidebar + main + detail panel pattern
- [x] Design dashboard and settings layout variations
- [x] Document overflow, sticky header, and scroll-region behavior
- [x] Document density guidance for compact vs standard desktop usage
