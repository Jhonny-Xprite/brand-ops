## Story
**Como** time responsável pela evolução do produto,  
**Eu quero** reconciliar o novo wireframe com o código e as superfícies já implementadas,  
**Para que** as próximas refatorações sigam uma única arquitetura real e não criem fluxos duplicados.

## Acceptance Criteria
1. Existe uma matriz explícita relacionando wireframe, rota atual, componente atual e decisão arquitetural
2. A Creative Library é tratada como superfície real já implementada, não como detalhe opcional
3. Fica definido se a experiência de mídia continuará em `/creative-library`, migrará para `/projeto/[id]/media`, ou será compartilhada entre os dois contextos
4. O documento final identifica quais telas do projeto atual são canônicas, quais são placeholders e quais precisam ser absorvidas ou descartadas
5. A saída da story é repo-first e pode guiar implementação sem nova rodada de descoberta

## Tasks
- [x] Mapear wireframe novo para as rotas reais do app
- [x] Inventariar superfícies existentes no fluxo Home -> Projeto -> Creative Library
- [x] Classificar cada superfície como canônica, placeholder, duplicada ou divergente
- [x] Registrar decisão arquitetural para a Creative Library no contexto do produto
- [x] Publicar artefato repo-first de reconciliação e atualizar o backlog do Epic 0
