## Story
**Como** usuario navegando entre areas do projeto,  
**Eu quero** um shell consistente que preserve o contexto ativo e a navegacao entre as superficies do produto,  
**Para que** eu nao perca o projeto selecionado nem precise reconstruir meu fluxo a cada troca de pagina.

## Acceptance Criteria
1. A top bar global mostra o projeto ativo e oferece acao clara para trocar de projeto
2. A navbar horizontal expõe as areas canonicas do projeto e destaca o item ativo
3. O contexto do projeto permanece estavel via rota dinamica e estado compartilhado
4. O shell funciona corretamente nas paginas principais do produto sem recarregamentos abruptos
5. O layout respeita a baseline desktop definida no Epic 0
6. `lint`, `typecheck`, `build` e `test` passam no fechamento da story
