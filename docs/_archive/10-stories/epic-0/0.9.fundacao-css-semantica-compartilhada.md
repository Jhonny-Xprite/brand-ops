## Story
**Como** desenvolvedor mantendo o design-system real do produto,  
**Eu quero** completar as classes semanticas e aliases compartilhados ja usados pelo app,  
**Para que** as superficies existentes renderizem com consistencia e sem depender de estilos implicitos ou orfaos.

## Acceptance Criteria
1. Todas as classes semanticas ja usadas em superficies reais do app tem definicao compartilhada em `globals.css` ou equivalente oficial
2. Tokens e aliases necessarios ao sistema atual estao alinhados com `tailwind.config.js`
3. O app nao depende de classes semanticas "fantasma" para paineis, dialogs, campos e shells desktop
4. As classes adicionadas permanecem compativeis com as superficies ja implementadas, especialmente Home, Project Shell e Creative Library
5. Lint, typecheck, build e tests passam apos a fundacao compartilhada ser completada

## Tasks

- [x] Validar compatibilidade com Home, Project Shell, Config e Creative Library
