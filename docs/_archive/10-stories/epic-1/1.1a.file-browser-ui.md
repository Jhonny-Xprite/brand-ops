## Story

**As a** creative operator,  
**I want** the creative library shell with ingest, browse layout, and baseline browse controls,  
**so that** the app has a stable host surface for metadata editing and later versioning work

---

## Acceptance Criteria

1. The creative library screen renders the baseline file-browser layout with sidebar shell, top controls shell, main file area, and details panel shell.
2. Users can ingest supported files from the OS into the approved local storage root `E:\BRAND-OPS-STORAGE\`, with successful ingest creating persisted `CreativeFile` records under that contract.
3. Grid/list view toggle works and persists in local storage.
4. Baseline browse controls exist for filename search, type/status filtering, and sort order, without expanding into the full dedicated search epic scope.
5. Thumbnail handling supports images automatically, falls back safely for unsupported preview types, and does not break the file list.
6. Multi-select is supported for baseline file-card/list interactions described in Epic 1.
7. Keyboard navigation and focus behavior cover the primary browse interactions on the shell delivered by this story.
8. User-visible error handling for upload and filesystem failures follows `docs/error-handling-spec.md`.
9. The story does not implement git-backed version history, richer folder-navigation ergonomics, or advanced file actions yet; it prepares the host surface and ingest path for later Epic 1 stories.
10. Lint, typecheck, tests, and build pass after the implementation.
