## Story

**As a** creative operator,  
**I want** to inspect version history for a file with metadata context,  
**so that** I can understand what changed before deciding whether to compare or restore an older version

---

## Acceptance Criteria

1. The version history viewer uses `FileVersion` as the primary local timeline source.
2. The viewer shows version number, date, commit message, and commit hash for each entry.
3. Users can open a version entry and inspect the corresponding file/metadata snapshot behavior defined by the Epic 1 PRD.
4. The viewer supports the baseline compare/view interactions planned for Epic 1.
5. History refresh and unavailable-history behavior follow `docs/error-handling-spec.md` and `docs/git-integration-flow.md`.
6. Keyboard behavior for navigating the history list is implemented for the primary interactions.
7. The implementation remains compatible with the later rollback flow instead of tightly coupling history rendering to one-off UI state.
8. Lint, typecheck, tests, and build pass after the implementation.
