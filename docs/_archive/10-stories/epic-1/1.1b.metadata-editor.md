## Story

**As a** creative operator,  
**I want** to edit file metadata inline from the creative library,  
**so that** type, status, tags, and notes can be maintained without leaving the file workflow

---

## Acceptance Criteria

1. The metadata editor uses the final `CreativeFile` / `FileMetadata` contract established by `PRE-1.1`.
2. Users can edit type, status, tags, and notes from the creative library without leaving the browsing context.
3. Validation behavior follows the messages and UI patterns from `docs/error-handling-spec.md`.
4. Type and status are required, tags honor the agreed limit, and notes honor the 500-character limit.
5. Tag editing preserves the app contract `tags: string[]`, not the raw persisted storage shape.
6. The editor supports tag suggestion/history behavior that stays within the approved app contract rather than inventing a second persisted tag model.
7. Save behavior updates the visible file state and persists the metadata in SQLite.
8. Keyboard support covers tab, shift-tab, enter/save, and escape/cancel for the editor flow.
9. The implementation is ready for later auto-versioning rather than inventing its own persistence side path.
10. Lint, typecheck, tests, and build pass after the implementation.
