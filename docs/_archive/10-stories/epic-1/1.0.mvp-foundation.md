## Story

**As a** product team,
**I want** to validate the core upload → edit → persist → reload flow end-to-end,
**so that** we know the foundation is solid before investing 112 more hours in versioning features

## Rationale

This story is a **validation checkpoint** before Stories 1.1a-1.5. It:
- Tests database connection, file I/O, state management, API structure
- Proves the "no-git" workflow (upload + metadata edit) works
- Fails fast if fundamentals are broken (before 120h invested)
- Gives 1.1a-1.5 a proven foundation to build on

---

## Acceptance Criteria

1. **File Upload**: User can upload 1 file via file input → saved to `E:\BRAND-OPS-STORAGE\{filename}`
2. **Database Create**: `CreativeFile` record created in SQLite with correct path, filename, type (auto-detected)
3. **Immediate Visibility**: File appears in file browser list within 1 second after upload
4. **Metadata Form**: Sidebar form shows fields: Type, Status, Tags, Notes (all editable)
5. **Metadata Edit**: User can edit any field and click Save → metadata persists to `FileMetadata` table
6. **UI Refresh**: File list updates immediately after metadata save (visual feedback)
7. **Type Required**: Type field is required; saving without it shows error: "Type is required"
8. **Status Required**: Status field is required; saving without it shows error: "Status is required"
9. **Persistence on Reload**: Close app completely → Reopen → File + metadata are still there exactly as saved
10. **Multiple Edits**: Edit same file 3 times → all changes persist (no data loss)
11. **No Console Errors**: Zero JavaScript errors in browser console during entire flow
12. **Build Success**: `npm run lint`, `npm run typecheck`, `npm test`, `npm run build` all pass

## Technical Scope

### Frontend Components

```
CreativeLibrary (page)
  ├─ FileUploadInput
  │   ├─ Input type="file" accept all
  │   ├─ onChange → POST /api/files/upload
  │   └─ Show upload progress
  │
  ├─ FileList (simple table or list)
  │   ├─ Map CreativeFile[] from state
  │   ├─ Show: filename, type (if set), status (if set)
  │   ├─ Click row → select file, open sidebar
  │   └─ Show: "0 files" if empty, "1 file" etc.
  │
  └─ MetadataForm (right sidebar)
      ├─ TypeSelect (dropdown: image, video, carousel, document, other)
      ├─ StatusSelect (dropdown: Draft, In Review, Approved, Done)
      ├─ TagsInput (free-form text, comma-separated)
      ├─ NotesTextarea (max 500 chars, show counter)
      ├─ [Save] button (disabled until changes detected)
      ├─ [Cancel] button
      └─ Error messages (inline, red text)
```
