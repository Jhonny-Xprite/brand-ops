## 🧭 Princípios da Voz

A voz do Brand-Ops é o som da ordem. Ela não pede permissão; ela abre o caminho.

| Princípio | Descrição | O que evoca no usuário |
| :--- | :--- | :--- |
| **Incisiva** | Direta ao ponto, sem advérbios desnecessários ou "por favor". | Decisividade. |
| **Autoritária** | O sistema sabe o que é melhor para a marca. Ele impõe o padrão. | Confiança técnica. |
| **Facilitadora** | A linguagem serve para acelerar a ação, não para explicar o óbvio. | Fluidez soberana. |

---

## 🚫 Dicionário de Expurgos (Anarquia vs. Comando)

Banimos o vocabulário do Caos de Pastas para evitar a contaminação mental.

| Termo Proibido | Termo de Comando (O Zag) | Por que mudar? |
| :--- | :--- | :--- |
| Criar Pasta | **Iniciar Contexto / Protocolo** | "Pasta" é bagunça. "Contexto" é estratégia. |
| Gerenciar Arquivos | **Comandar Ativos** | Gerentes olham. Comandantes agem. |
| Upload de Mídia | **Ingerir Matéria-Prima** | O arquivo está entrando em um processo digestivo estratégico. |
| Configurações | **Motores do Sistema** | Configurações são chatas. Motores são potentes. |
| Favoritos | **Ativos de Elite** | "Favoritos" é subjetivo. "Elite" é meritocracia de performance. |

## ✍️ Guia de Micro-copy UX

### 🔘 Botões de Ação
- **Errado**: "Salvar Alterações", "Clique aqui", "Submeter".
- **Comando**: "Consolidar", "Ativar", "Disparar Protocolo".

### 📥 Uploads e Ingestões
- **Contexto**: O usuário arrasta um arquivo bruto.
- **Micro-copy**: "Analisando Contexto...", "Identificando Ativo...", "Soberania Garantida".

### ⚠️ Erros do Usuário
Não pedimos desculpas por erros técnicos de outros. Somos assertivos.
- **Errado**: "Desculpe, o arquivo é grande demais. Tente novamente."
- **Comando**: "Limite Operacional Excedido. Reduza o Ativo e Re-tente."

### 📂 Estados Vazios (Empty States)
Onde o Caos costuma nascer.
- **Mensagem**: "Seu cockpit está limpo. Inicie um Protocolo para combater o caos."
