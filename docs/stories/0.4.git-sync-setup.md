## Dev Notes

### Relevant Architecture Context

[Source: architecture/storage-sync-strategy.md]
- Local storage: E:\BRAND-OPS-STORAGE\ (local SSD for primary access)
- Google Drive backup: /Brand-Ops-Backup/ (remote destination)
- Sync frequency: Daily at 23:00 UTC
- Conflict resolution: Local wins (local version overwrites remote if conflict)
- Metadata tracking: .sync-metadata/ folder

[Source: prd/epic-4-sync-versioning.md§Feature 4.1, 4.2]
- Git integration: Automatic commits on file changes
- Version tracking: .sync-metadata/  files store sync state
- Lock mechanism: .sync-locks/ files prevent concurrent edits
- rclone bi-directional sync: Keeps remote backup in sync

[Source: architecture/coding-standards.md§Git & Commit Standards]
- Conventional commits: `feat:`, `fix:`, `docs:`, `chore:`
- Reference story: `[Story X.Y]` in commit message
- Atomic commits: One logical change per commit
- Use `.gitignore` to exclude: environment, build, node_modules

### Project Structure

```
project-root/
├── .git/                    # Git repository (created in this story)
├── .gitignore              # Git exclusions (created in this story)
├── docs/
│   └── rclone-config.md    # rclone documentation (created in this story)
└── E:\BRAND-OPS-STORAGE\   # Local storage location
    ├── brand-ops.db        # SQLite database (from Story 0.1)
    ├── .sync-metadata/     # Sync state tracking (created in this story)
    └── .sync-locks/        # Concurrency control (created in this story)
```

### rclone Configuration Overview

```
Source:      E:\BRAND-OPS-STORAGE\
Destination: Google Drive /Brand-Ops-Backup/
Direction:   Bi-directional
Strategy:    Local wins on conflict
Schedule:    Daily 23:00 UTC (Windows Task Scheduler)
Retry:       3 attempts on failure
```

### Git Workflow

1. Developer works on files locally
2. Git tracks changes automatically (Story 2.x)
3. Commit messages reference stories (Story 2.x)
4. rclone daemon syncs daily at 23:00 UTC
5. Google Drive has backup copy
