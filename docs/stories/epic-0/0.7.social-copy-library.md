# Story 0.7: Estrutura da Social Assets e Copy Messaging

## Status
Ready

## Story
**Como** um social media manager,
**Eu quero** uma área para organizar peças prontas por canal e um depósito de textos de alta performance,
**Para que** eu possa postar conteúdos consistentes e com a copy validada.

## Acceptance Criteria
1. **Social Assets Grid:** Visualização organizada por formatos (Instagram 1:1, Stories 9:16, LinkedIn). Os cards devem mostrar a proporção do arquivo.
2. **Copy Depósito:** Interface focada em texto para armazenar Headlines, Scripts de VSL e legendas de Ads.
3. **Filtros por Ângulo:** Capacidade de filtrar copies por "Ângulo de Venda" ou "Público-Alvo".
4. **Menus de Navegação:** Menus "Social" e "Textos" na Navbar devem levar a essas áreas.
5. **Cópia Rápida:** Botão de "Copiar para área de transferência" em todos os blocos de texto.

## Tasks
- [ ] Criar rotas `/projeto/[id]/social` e `/projeto/[id]/copy`
- [ ] Implementar `SocialGrid` com detecção automática de proporção (aspect-ratio)
- [ ] Criar componente `CopyCard` com botão de Copy-to-Clipboard
- [ ] Implementar filtros de taxonomia (Ângulo/Avatar) para as copies
- [ ] Validar fluxo de "Curadoria" (Marcar asset como pronto para social)
