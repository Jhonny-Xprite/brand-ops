## Story
**Como** time evoluindo o hub de projetos e o shell principal,  
**Eu quero** migrar as superficies de projeto para a mesma linguagem semantica e reutilizavel ja presente no sistema real,  
**Para que** Home, layout de projeto e Creative Library deixem de parecer produtos diferentes.

## Acceptance Criteria
1. As superficies de projeto prioritarias usam tokens semanticos e componentes compartilhados aprovados
2. Home, cards, listagem, top bar, navbar e feedback states deixam de depender do sistema visual ad hoc claro
3. Reuso de `BrandLogo`, `MotionButton`, `StatusNotice`, `DialogShell` e outros componentes ocorre quando houver aderencia real de fluxo
4. A migracao reduz drift visual sem inventar um terceiro sistema paralelo
5. Lint, typecheck, build e tests passam nas superficies migradas

## Tasks
- [x] Migrar a Home Global para a baseline semantica compartilhada
- [x] Migrar componentes de Projects para tokens semanticos reais
- [x] Migrar layout de projeto para a mesma fundacao visual
- [x] Reusar componentes compartilhados onde houver aderencia real
- [x] Atualizar testes e documentacao da migracao
