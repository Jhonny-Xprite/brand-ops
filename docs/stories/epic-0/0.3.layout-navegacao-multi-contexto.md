# Story 0.3: Layout de Navegação Multi-Contexto

## Status
Ready

## Story
**Como** um usuário navegando no sistema,
**Eu quero** que a interface reflita o projeto que selecionei e me dê acesso rápido às suas 7 áreas,
**Para que** eu nunca perca o contexto do cliente em que estou trabalhando.

## Acceptance Criteria
1. **TopBar Global:** Deve exibir `[Logo BO] + Nome do Projeto Ativo` e um botão claro de "Trocar Projeto" (que volta para a Home).
2. **Navbar Horizontal:** Barra de segundo nível com os 7 menus traduzidos: Dashboard, Estratégia, Mídia, Social, Produção, Textos, Config.
3. **Menu Ativo:** O item da Navbar correspondente à página atual deve ter destaque visual (Violeta/Ouro).
4. **Persistência de Contexto:** O ID do projeto deve ser mantido na URL ou no estado global (Redux) ao navegar entre as abas.
5. **Responsividade Desktop:** Layout deve se comportar corretamente em diferentes tamanhos de monitores (Compacto a Wide).

## Tasks
- [ ] Criar componente `GlobalTopBar` com indicador de projeto
- [ ] Criar componente `ProjectNavbar` (Horizontal) com os 7 itens
- [ ] Implementar `ProjectLayout` (Wrapper) que engloba as páginas internas
- [ ] Configurar rotas dinâmicas: `/projeto/[id]/dashboard`, `/projeto/[id]/creative`, etc.
- [ ] Adicionar lógica de "Trocar Projeto" limpando o estado de contexto
