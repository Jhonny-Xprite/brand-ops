# Story 0.6: Estrutura da Strategy e Media Library

## Status
Ready

## Story
**Como** um estrategista de marca,
**Eu quero** espaços dedicados para arquivos de fundamentação estratégica e arquivos de mídia brutos,
**Para que** eu possa separar o que é "estudo/matéria-prima" do que é "produto final de anúncio".

## Acceptance Criteria
1. **Strategy Library:** Tela para visualizar documentos de estratégia (ex: Funis, Avatares, Ofertas). Deve suportar listagem de arquivos PDF/JSON e metadados de campanha.
2. **Media Library:** Navegador de arquivos focado em "RAW Assets". Deve suportar navegação por pastas (ex: /Ensaios, /Lifestyle) e exibição de fotos brutas.
3. **Isolamento de Diretório:** Arquivos da Strategy e Media devem ser salvos em subpastas específicas dentro de `E:\BRAND-OPS-STORAGE\[projeto]\`.
4. **Menus de Navegação:** Os menus "Estratégia" e "Mídia" na Navbar devem levar a essas respectivas áreas.
5. **Busca Contextual:** A busca dentro dessas áreas deve retornar apenas ativos pertencentes à respectiva biblioteca.

## Tasks
- [ ] Criar rotas `/projeto/[id]/strategy` e `/projeto/[id]/media`
- [ ] Implementar `FolderNavigator` para a Media Library
- [ ] Criar visualização de "Strategy Blocks" (Cards de Oferta/Campanha)
- [ ] Configurar lógica de armazenamento isolado por biblioteca no File System
- [ ] Adaptar o `FileList` para comportamentos específicos de Mídia Bruta
