## Story

**As a** design and development team,  
**I want** a shared foundation of semantic tokens and desktop UI rules,  
**so that** the MVP can be implemented consistently without inventing styles during Epic 1

---

## Acceptance Criteria

- [x] Foundation tokens are documented for color, typography, spacing, radius, borders, and elevation
- [x] Token naming uses semantic terms that can map cleanly to Tailwind and shadcn/ui usage
- [x] The visual direction is functional and minimalist, aligned with the project brief
- [x] Token guidance is desktop-first and does not require mobile-first rules
- [x] Color contrast guidance supports WCAG AA expectations
- [x] Repo-based foundation guide and implementation notes are prepared for handoff

---

## Tasks / Subtasks

- [x] Define semantic color tokens for surfaces, text, borders, focus, and status states
- [x] Define typography roles for page title, section heading, body, label, helper text, and monospace values
- [x] Define spacing, radius, border, and elevation scales
- [x] Document default interaction states: default, hover, focus, disabled, error, loading
- [x] Consolidate foundation tokens and usage rules in the repo handoff guide
- [x] Create implementation mapping notes for `@dev` without requiring production code delivery in this story

### Foundation Guidance

- Prefer a restrained blue-accent plus neutral-surface baseline over premium/luxury palettes
- Tokens must be semantic and implementation-oriented, for example `surface-default`, `text-muted`, `border-subtle`, `status-success`
- Foundation work here should support implementation stories; it should not redefine product branding
