## Story

**As a** creative operator,  
**I want** every meaningful change to create immutable local version history automatically,  
**so that** uploads, metadata edits, and replacements become traceable and recoverable

---

## Acceptance Criteria

1. Auto-versioning follows the approved architecture in `docs/git-integration-flow.md`.
2. Before the first versioning action runs, the separate creative storage repository rooted in `E:\BRAND-OPS-STORAGE\` is created or verified according to `docs/git-integration-flow.md`.
3. Upload, metadata save, and file replacement create the correct commit categories and `FileVersion` records.
4. Commit generation uses deterministic messages and sequential per-file version numbers.
5. Eligible metadata saves batch within the approved batch window instead of producing unnecessary commit noise.
6. Retry and terminal-failure behavior follow the approved `PRE-1.3` and `PRE-1.5` semantics.
7. The implementation emits reliable versioning lifecycle state for later UX stories instead of burying queue-state behavior in one-off UI code.
8. The implementation reads and writes against the final `CreativeFile` / `FileMetadata` / `FileVersion` schema.
9. Lint, typecheck, tests, and build pass after the implementation.
