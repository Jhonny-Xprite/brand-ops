## Story

**As a** creative operator,  
**I want** the creative library to support richer navigation and browse ergonomics,  
**so that** the Epic 1 library feels efficient before versioning is layered on top

---

## Acceptance Criteria

1. The library adds the approved navigation shell behavior still required by Epic 1, including breadcrumb-oriented navigation, richer keyboard flow, and browse ergonomics not covered by `1.1a`.
2. The implementation builds on the stabilized browser/editor integration from `1.1a` and `1.1c` instead of reworking the earlier ingest flow.
3. Contextual file actions required by the Epic 1 docs are implemented or surfaced in the approved browse experience.
4. Rename, duplicate, properties, and related browse actions use the approved filesystem and error semantics already defined in project docs.
5. Accessibility and keyboard behavior extend the creative-library surface without regressing the earlier browser shell.
6. The result leaves a stable pre-versioning interaction surface for `1.2`.
7. Lint, typecheck, tests, and build pass after the implementation.
