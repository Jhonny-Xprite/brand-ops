## Story

**As a** creative operator,  
**I want** to restore one or many files to older versions safely and instantly,  
**so that** version history becomes actionable instead of read-only

---

## Acceptance Criteria

1. Rollback follows the approved path from `docs/git-integration-flow.md` and `docs/concurrent-editing-strategy.md`.
2. Restoring an older version creates a new versioned state instead of deleting history.
3. The confirmation flow is explicit and prevents ambiguous destructive actions.
4. Rollback preserves the immutable version chain and records the correct rollback commit message.
5. Rollback failure, file-lock, and corrupted-source behavior follow the approved `PRE-1.3` semantics.
6. The UI refreshes current file state and version history after successful rollback.
7. Batch rollback for multiple selected files is supported for the Epic 1 MVP path and preserves the same immutable-history guarantees per file.
8. Lint, typecheck, tests, and build pass after the implementation.
