## Story

**As a** team shipping Epic 1 incrementally,  
**I want** the file browser and metadata editor hardened together before git versioning starts,  
**so that** the UI and persistence baseline are stable before more complex history features are added

---

## Acceptance Criteria

1. The file browser and metadata editor work together as one coherent creative library flow.
2. Upload, browse, select, and metadata-edit operations update the screen consistently without requiring page reload.
3. The integration covers the primary no-git journey: upload -> browse -> edit metadata -> reload -> confirm persisted state.
4. Search, filter, and view-state behavior remain correct after metadata edits.
5. Error behavior remains aligned with `docs/error-handling-spec.md`.
6. The integrated flow is covered by tests at the level appropriate for the current codebase baseline.
7. The implementation leaves a stable base for story `1.2` auto-versioning rather than duplicating or bypassing state changes.
8. Lint, typecheck, tests, and build pass after the implementation.
