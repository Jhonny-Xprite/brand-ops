## Dev Notes

### Relevant Architecture Context

[Source: architecture/source-tree.md]
- Folder structure: `src/components/{atoms,molecules,organisms}` (atomic design)
- Pages: `src/pages/` (pages router) or `src/app/` (app router)
- Store: `src/store/{api,slices}`
- Types: `src/lib/types/`

[Source: architecture/coding-standards.md§TypeScript Standards]
- No `any` types
- Strict mode required
- All data models have TypeScript interfaces
- Type all function parameters and return values

[Source: architecture/tech-stack.md§Frontend Stack]
- Next.js 14+
- React 18+
- TypeScript (strict mode)
- Redux Toolkit + RTK Query
- Tailwind CSS v3
- shadcn/ui

### Project Structure

```
src/
├── components/
│   ├── atoms/          # Base components (Button, Input, Label, Badge, etc)
│   ├── molecules/      # Simple combinations (FormField, Card, Header, Footer)
│   └── organisms/      # Complex UI sections (Sidebar, MainLayout, FileExplorer)
├── pages/ or app/      # Page routes (empty structure, ready for pages)
├── lib/
│   ├── types/
│   │   ├── models.ts   # Data model interfaces (Creative, Metadata, Version, SyncMetadata)
│   │   └── index.ts    # Export all types
│   ├── utils/          # Helper functions
│   └── prisma.ts       # Prisma Client singleton (created in Story 0.1)
├── store/
│   ├── index.ts        # Redux store configuration
│   ├── api/            # RTK Query endpoints
│   └── slices/         # Redux slices (empty, ready for implementation)
├── hooks/              # Custom React hooks
└── styles/             # Global styles + design tokens
```

### Atomic Design Pattern

[Source: architecture/front-end-spec.md§Atomic Design]
- **Atoms:** Smallest components (Button, Input, Label, Badge, Icon)
- **Molecules:** Simple combinations (Form fields, Cards, Headers)
- **Organisms:** Complex sections (Sidebars, Layouts, Dialogs)
- **Templates:** Page layouts (ready in Story 0.2 structure)
- **Pages:** Specific instances (implemented in later stories)

### Previous Story Context
Story 0.1 (Prisma Setup) provides database schema. Story 0.2 creates TypeScript interfaces that mirror that schema.

### Testing Standards

[Source: architecture/coding-standards.md§Testing Standards]
- Test framework: Jest
- Test location: `src/**/__tests__/*.test.ts`
- For this story: Type definition tests (verify interfaces are correct)
- Coverage target: >70%
