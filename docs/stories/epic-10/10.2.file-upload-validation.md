## Story

**As a** security-conscious developer,  
**I want** file uploads to be validated for type and size,  
**so that** the application rejects invalid files before processing.

---

## Acceptance Criteria

1. [ ] File size limit enforced
   - [ ] Environment variable `MAX_FILE_SIZE_MB` added to `.env.example`
   - [ ] File size checked in `src/pages/api/files/` routes
   - [ ] Returns 413 Payload Too Large on oversized files
   - [ ] Default limit: 50MB

2. [ ] File type validation enforced
   - [ ] Environment variable `ALLOWED_FILE_TYPES` added to `.env.example`
   - [ ] MIME type whitelist configured (jpg, png, mp4, mov, psd, etc)
   - [ ] Returns 400 Bad Request for invalid file types
   - [ ] Default types: jpg, jpeg, png, gif, mp4, mov, webm, psd, ai

3. [ ] Validation happens server-side
   - [ ] All upload endpoints validate
   - [ ] Consistent error messages
   - [ ] No bypasses via direct API calls

4. [ ] Documentation updated
   - [ ] `.env.example` includes file upload config
   - [ ] API documentation explains file constraints
   - [ ] Error messages are user-friendly

5. [ ] Tests pass
   - [ ] Test oversized file rejection
   - [ ] Test invalid file type rejection
   - [ ] Test valid files accept

## Dev Notes

- File validation should happen in API route handler
- Use MIME type whitelist, not blacklist (security best practice)
- Check file extension AND MIME type (some can be spoofed)
- Use the `busboy` library already in project for stream validation
- Error messages should be user-friendly but not reveal security details
