## Story

**As a** developer,  
**I want** linting and type-checking to run automatically before commits,  
**so that** bad code never enters the repository.

## Configuration Examples

**package.json:**
```json
{
  "devDependencies": {
    "husky": "^8.0.0",
    "lint-staged": "^15.0.0"
  },
  "lint-staged": {
    "*.{ts,tsx}": [
      "eslint --fix",
      "tsc --noEmit"
    ]
  }
}
```

**.husky/pre-commit:**
```bash
#!/bin/sh
. "$(dirname "$0")/_/husky.sh"

npx lint-staged
```

## Windows/WSL Compatibility

**Important for Windows developers using WSL:**

- Hook scripts run in WSL environment (uses POSIX shell)
- Path conversions between Windows and WSL are automatic in most cases
- **Test before committing:**

  ```bash
  # Windows Command Prompt or PowerShell
  echo "test" > test.ts
  git add test.ts
  git commit -m "test"  # Should trigger hook in WSL
  ```

- If hooks don't run, verify git is configured to use WSL git: `git config core.autocrlf`
- On native Windows (not WSL), ensure shebang is preserved: `git config core.safecrlf false`

## What Happens on Commit

1. Developer runs: `git commit -m "message"`
2. Pre-commit hook triggers
3. lint-staged runs on changed files:
   - ESLint checks syntax/style
   - TypeScript checks types
4. If errors:
   - Commit blocked with error message
   - Developer fixes issues
   - Retries commit
5. If clean:
   - Commit proceeds normally

## Bypass (Rarely Needed)

If developer needs to skip hooks in emergency:
```bash
git commit --no-verify -m "emergency commit"
```

**Note:** This should only be used for critical hotfixes and must be reviewed later
