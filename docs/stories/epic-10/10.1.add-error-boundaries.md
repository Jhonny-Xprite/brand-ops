## Dev Notes

- Use React ErrorBoundary class component (not functional)
- Reference: https://react.dev/reference/react/Component#catching-rendering-errors-with-an-error-boundary
- Consider showing different fallback UI for development vs production
- **Error Logging Strategy:** Implement graceful fallback for error logging:
  - Primary: Send to centralized logging service (if available)
  - Fallback 1: Log to browser console (dev environment)
  - Fallback 2: Store in sessionStorage (if network unavailable)
  - Fallback 3: Silent fail (never break the error boundary with logging errors)
- Error boundary should NOT catch:
  - Event handlers (use try/catch instead)
  - Asynchronous code (use Promise.catch instead)
  - Server-side rendering
